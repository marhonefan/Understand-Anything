# Understand Anything 安装与更新指南

本文档汇总 Understand Anything 的各种安装方式与更新方式,覆盖**本地源码安装**(开发/定制场景)、**官方 marketplace 安装**(普通使用)以及**多平台安装**(Codex / Cursor / Gemini 等)。

> 当前插件版本:**2.8.1**

---

## 环境要求

| 依赖 | 版本 | 说明 |
|------|------|------|
| Node.js | **>= 22**(开发于 v24) | 低于 22 无法构建/运行 |
| pnpm | **>= 10** | 由根 `package.json` 的 `packageManager` 字段锁定 |
| Claude Code(或其它受支持平台) | 最新版 | 见下方平台兼容表 |

> ⚠️ **关键认知**:Claude Code 安装插件时是把源码目录**原样拷贝**到缓存,**不会自动编译 TypeScript**。因此任何「从源码安装」的方式,都必须**先构建再安装**,否则 `/understand-chat`、`/understand-diff`、`/understand-explain`、`/understand-onboard` 等依赖编译产物(`dist/`)的命令会失效。

---

## 方式一:从本地源码安装到 Claude Code(推荐用于开发/定制)

适用于你已经把仓库 clone 到本地、想用本地这份代码(可能含自己的改动)安装的场景。

仓库根目录已带 `.claude-plugin/marketplace.json`,因此可把**本地目录直接当作 marketplace** 安装。

### 步骤

```bash
# 1) 进入仓库根目录
cd /path/to/Understand-Anything

# 2) 安装依赖
pnpm install

# 3) 构建 core 与 skill 两个包(必做,否则编译产物缺失)
pnpm --filter @understand-anything/core build
pnpm --filter @understand-anything/skill build
```

构建产物落点:
- `understand-anything-plugin/packages/core/dist/` —— 共享分析引擎
- `understand-anything-plugin/dist/` —— skill 源码(`src/` → `dist/`,`tsconfig.json` 中 `outDir: "dist"` / `rootDir: "src"`)

```bash
# 4) 在 Claude Code 对话框中输入(以下两条为斜杠命令,需在 Claude Code 内执行):
/plugin marketplace add /path/to/Understand-Anything
/plugin install understand-anything
```

```text
# 5) 重开一个新的 Claude Code 会话(旧会话缓存了旧 prompt,不重开不生效)

# 6) 在目标项目目录运行:
/understand
```

> 首次 `/understand` 会全量分析整个代码库,较费 token;跑完自动打开 dashboard。之后默认增量,只重分析变更文件。

### 验证安装成功

- 新会话中输入 `/plugin`,确认 `understand-anything` 已启用;或
- 直接输入 `/understand`,有命令提示即说明已装好。

## 方式二:本地改动后重新同步(开发迭代)

已通过方式一/官方 marketplace 装过,之后修改了源码,**无需重装 marketplace**,直接覆盖缓存即可。Claude Code 的插件缓存路径:

```
~/.claude/plugins/cache/understand-anything/understand-anything/<版本号>/
```

```bash
# 1) 重新构建
pnpm --filter @understand-anything/core build
pnpm --filter @understand-anything/skill build

# 2) 覆盖缓存(把 <VERSION> 换成当前已安装版本,如 2.8.1)
cp -R ./understand-anything-plugin/* \
  ~/.claude/plugins/cache/understand-anything/understand-anything/<VERSION>/

# 3) 重开会话生效
```

> 查看当前缓存的版本号:
> ```bash
> ls ~/.claude/plugins/cache/understand-anything/understand-anything/
> ```
>
> ⚠️ 不要用符号链接(symlink)代替拷贝 —— Claude Code 的 Search/Glob 工具无法跟随符号链接。

### 恢复为官方版本

卸载后从 marketplace 重新安装,缓存会从上游仓库重新填充:

```text
/plugin uninstall understand-anything
/plugin install understand-anything
```

## 方式三:从官方 marketplace 安装(普通使用,最简单)

不需要本地源码、只想正常使用时:

```text
/plugin marketplace add Egonex-AI/Understand-Anything
/plugin install understand-anything
```

### 更新到最新官方版

```text
/plugin marketplace update understand-anything
/plugin install understand-anything
```

> 官方版由 marketplace 提供已构建好的产物,无需本地 `pnpm build`。

## 方式四:其它平台安装(Codex / Cursor / Gemini CLI / OpenCode 等)

非 Claude Code 平台使用仓库自带的 `install.sh`(Windows 用 `install.ps1`)。

### 一行安装(默认从官方仓库 clone)

**macOS / Linux:**
```bash
curl -fsSL https://raw.githubusercontent.com/Egonex-AI/Understand-Anything/main/install.sh | bash
# 或直接指定平台,跳过交互选择:
curl -fsSL https://raw.githubusercontent.com/Egonex-AI/Understand-Anything/main/install.sh | bash -s codex
```

**Windows(PowerShell):**
```powershell
iwr -useb https://raw.githubusercontent.com/Egonex-AI/Understand-Anything/main/install.ps1 | iex
```

安装器会把仓库 clone 到 `~/.understand-anything/repo`,并为所选平台创建对应符号链接。安装后重启 CLI/IDE。

### 使用本地已有的 clone(而非重新 clone)

通过 `UA_DIR` 环境变量指向你本地的仓库目录:

```bash
UA_DIR=/path/to/Understand-Anything ./install.sh codex
```

> 可用平台值:`gemini`、`codex`、`opencode`、`pi`、`openclaw`、`antigravity`、`vibe`、`vscode`、`hermes`、`cline`、`kimi`、`trae`、`nanobot`、`kiro`

### 更新与卸载

```bash
./install.sh --update              # 拉取最新代码(符号链接自动指向新版本)
./install.sh --uninstall <平台>    # 移除指定平台的链接
```

### Cursor / VS Code + Copilot 自动发现

- **Cursor**:clone 本仓库后通过 `.cursor-plugin/plugin.json` 自动发现,无需手动安装。
- **VS Code + GitHub Copilot(v1.108+)**:通过 `.copilot-plugin/plugin.json` 自动发现。

### Copilot CLI

```bash
copilot plugin install Egonex-AI/Understand-Anything:understand-anything-plugin
```

## 升级版本号(维护者发版用)

向远端推送新版本时,需**同步**更新以下五个文件的 `version` 字段:

- `understand-anything-plugin/package.json`
- `understand-anything-plugin/.claude-plugin/plugin.json`
- `.claude-plugin/plugin.json`
- `.cursor-plugin/plugin.json`
- `.copilot-plugin/plugin.json`

> `.claude-plugin/marketplace.json` **不带** version 字段 —— `plugins[]` 条目仅支持 `name` 与 `source`,新增其它字段会导致 marketplace schema 校验失败。

## 常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| `/plugin marketplace add <本地路径>` 报错 | 平台不认本地路径 | 改用**方式二**:直接 `cp -R` 拷进缓存目录 |
| `/understand-chat` 等命令报错/无响应 | 未构建,`dist/` 缺失 | 重新执行 `pnpm --filter ... build` 后重装/重同步 |
| 改了代码但行为没变 | 旧会话缓存了旧 prompt | **重开新会话** |
| 改了代码但仍是旧逻辑(已重开会话) | 缓存未更新 | 重新构建 + 覆盖缓存(方式二) |
| 符号链接方式不生效 | Search/Glob 无法跟随 symlink | 用 `cp -R` 实拷,勿用 `ln -s` |
| 构建报原生模块错误(darwin/arm64 + Node 24) | 原生 `tree-sitter` binding 失败 | 本项目已改用 `web-tree-sitter`(WASM),确认依赖已正确安装 |
